# 参考文档索引

这一部分解释这些模板该怎么配合使用，而不是把它们当成一堆孤立文件。

## 全部参考文档

| 文档 | 内容 | 何时读 |
|------|------|--------|
| [`method-map.md`](./method-map.md) | 常见失败模式与对应修复工件 | 遇到反复失败时 |
| [`initializer-agent-playbook.md`](./initializer-agent-playbook.md) | 初始化阶段操作手册 | 首次初始化项目时 |
| [`coding-agent-startup-flow.md`](./coding-agent-startup-flow.md) | 每轮编码会话的固定流程 | 不确定开工流程时 |
| [`prompt-calibration.md`](./prompt-calibration.md) | 根指令文件的编写原则 | 调整根指令时 |
| [`planning-methodology.md`](./planning-methodology.md) | 功能拆解方法论 | 拆解功能或创建执行计划时 |
| [`review-checklist.md`](./review-checklist.md) | 代码审查清单 | 功能完成前的代码审查 |
| [`testing-strategy.md`](./testing-strategy.md) | 测试策略指南 | 编写测试或设计验证方案时 |

## 推荐阅读顺序

1. **`method-map.md`** — 理解常见陷阱和对应修复手段
2. **`initializer-agent-playbook.md`** — 知道第一轮会话要产出什么
3. **`coding-agent-startup-flow.md`** — 掌握每轮会话的开收工流程
4. **`prompt-calibration.md`** — 学会写恰到好处的根指令
5. **`planning-methodology.md`** — 学会拆解复杂任务
6. **`review-checklist.md`** — 功能完成前的质量把关
7. **`testing-strategy.md`** — 验证方案设计

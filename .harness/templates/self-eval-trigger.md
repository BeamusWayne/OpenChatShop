# 自我评审触发协议

## 触发时机

在自治模式下，以下事件触发自我评审：

1. 每个功能标记为 passing 之前
2. 每个功能标记为 blocked 之前
3. 连续完成 3 个功能后（整体健康检查）

## 评审流程

1. 读取 evaluator-rubric.md
2. 对刚完成的功能按六个维度打分
3. 如果总分 < 8（满分 12）：
   - 如果是 passing 功能 -> 降级为 in_progress，再迭代一次
   - 如果是 blocked 功能 -> 保持 blocked，记录评分详情
4. 如果总分 >= 8：
   - 确认 passing 状态
   - 记录评分到 claude-progress.md

## 评分校准提醒

自治模式下 agent 容易给自己打高分。缓解措施：
- 每个维度的 2 分要求明确的证据（不只是"看起来没问题"）
- 验证维度（维度 2）必须引用 auto_check 的实际输出
- 范围纪律维度（维度 3）要求列出本轮实际修改的文件列表

## 评审结果记录

每次评审的分数记录在 claude-progress.md 的自治迭代记录表中。

## 与质量文档的联动

每完成 3 个功能后，同时更新 evaluator-rubric.md 的质量快照。
